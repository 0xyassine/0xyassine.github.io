i searched for Adminer 4.6.2 4.7.6 exploit and found a useful link:
https://medium.com/bugbountywriteup/adminer-script-results-to-pwning-server-private-bug-bounty-program-fe6d8a43fe6f

how the attack work ?

1. Setting up MySQL server in kali.
2. Connecting adminer to the MySQL server.
3. Reading local files by read data local infile command and insert the result into a table.

Methodology

By default , MySql use port 3306 , but this port run locally , so to fix this problem i edited the config file of MySql and changed the port from 3306 to 3307 then i can forward connection comming to port 3306 from the adminer.php to port 3307 of MySql because adminer.php will connect to port 3306 by default.
Edit the config file : nano /etc/mysql/mariadb.conf.d/50-server.cnf ,remove the # before the port number and set it to 3307 then save it

this script is importing a make_archive function from shutil,and make_archive takes three variables , so if i hijack the python path and let it import my custom function, i can run os commands as root .
To do that , i need to import my python path to sudo environement .Thankfully this option is build in sudo command (-E, --preserve-env),for more info , you can run man sudo.

Note: i will create a hidden folder, because there is a rule in the cron job that delete all files in /tmp and /home/waldo each three minutes.

To verify , run sudo -u root /opt/scripts/admin_tasks.sh and select option 3
